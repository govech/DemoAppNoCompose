## 📝 文档维护

### 文档更新原则
- 代码变更时同步更新相关文档
- 新功能必须包含使用文档
- 重要决策必须记录 ADR
- 定期审查和更新过时文档

### 文档贡献
- 发现文档错误请提交 Issue
- 改进建议欢迎提交 Pull Request
- 新增文档请遵循现有格式
- 重要变更需要团队评审

### 文档规范
- 使用 Markdown 格式编写
- 中英文混排时注意空格
- 代码示例要完整可运行
- 图片和图表使用相对路径
